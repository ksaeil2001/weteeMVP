## 1. 개요

### 1.1 목적

이 문서는 과외 관리 통합 플랫폼 "WeTee"의 모든 화면과 사용자 경험을 정의합니다. 

**주요 목적**:
- 디자이너가 참고할 화면 설계 가이드 제공
- 개발자가 구현할 화면 구조와 인터랙션 명세
- 일관된 사용자 경험 보장

### 1.2 범위

**포함**:
- MVP 1단계 8개 기능(F-001~F-008)의 모든 화면
- 화면 간 전환 흐름
- 공통 UI 컴포넌트 및 디자인 시스템
- 인터랙션 패턴

**제외**:
- API 엔드포인트 (→ API_명세서.md 참조)
- 데이터베이스 스키마 (→ 데이터베이스_설계서.md 참조)
- 비즈니스 로직 세부사항 (→ 기능 명세서 F-001~F-008 참조)

### 1.3 설계 원칙

#### 원칙 1: 직관성
- 처음 사용하는 사용자도 5분 내에 핵심 기능 이해 가능
- 최소 3번의 탭으로 모든 주요 작업 완료
- 명확한 레이블과 아이콘 사용

#### 원칙 2: 일관성
- 동일한 작업은 동일한 방식으로 수행
- 통일된 색상, 타이포그래피, 간격
- 플랫폼별 가이드라인 준수 (iOS Human Interface Guidelines, Material Design)

#### 원칙 3: 효율성
- 반복 작업 최소화 (자동 입력, 기본값 제공)
- 중요 정보는 위에, 부가 정보는 아래
- 로딩 시간 최소화 및 스켈레톤 UI 사용

#### 원칙 4: 접근성
- 명도 대비 4.5:1 이상 (WCAG AA 준수)
- 최소 터치 영역 44x44pt (iOS), 48x48dp (Android)
- 스크린 리더 지원 (이미지 대체 텍스트)

## 3. 사용자 플로우

### 3.1 주요 시나리오별 User Flow

#### 🎯 시나리오 1: 첫 사용자 온보딩 (선생님)

```
S-001 스플래시
  ↓
S-002 온보딩 (3개 슬라이드)
  ↓
S-004 회원가입
  ↓ (이메일 인증)
S-003 로그인
  ↓
S-006 그룹 생성
  ↓
S-009 초대 코드 발급
  ↓
S-013 정규 수업 등록
  ↓
S-012 달력 메인 (홈)
```

**소요 시간**: 5~7분  
**화면 수**: 7개  
**주요 인터랙션**: 입력(4), 선택(2), 버튼 클릭(10)

---

#### 🎯 시나리오 2: 수업 후 기록 작성 (선생님)

```
S-012 달력 메인
  ↓ (오늘 수업 카드 탭)
S-019 출결 체크
  ↓ ("출석" 버튼 탭)
S-022 수업 기록 작성
  ↓ (작성 완료)
S-023 수업 기록 상세
  ↓ (완료)
S-041 알림 센터 (학생/학부모에게 자동 알림)
```

**소요 시간**: 2~3분  
**화면 수**: 4개  
**주요 인터랙션**: 출결 버튼 1회, 텍스트 입력 3~5회, 저장 1회

---

#### 🎯 시나리오 3: 월말 정산 (선생님 → 학부모)

```

  ↓

  ↓ (수업 선택 + 발송)

  ↓

S-031 결제 완료
  ↓

**소요 시간**: 5분 (선생님 2분 + 학부모 3분)  
**화면 수**: 7개  
**주요 인터랙션**: 선택(1), 버튼 클릭(5), 외부 앱(1)

## 4. 와이어프레임 설명

### 4.1 공통 레이아웃 구조

#### 기본 화면 구조

```
┌─────────────────────────────────────┐
│  ← 뒤로  |  제목 (중앙)  |  액션 →  │ ← 상단 네비게이션 (56pt)
├─────────────────────────────────────┤
│                                     │
│                                     │
│         메인 콘텐츠 영역              │
│         (스크롤 가능)                │
│                                     │
│                                     │
├─────────────────────────────────────┤
│  주요 액션 버튼 (선택)                │ ← 하단 고정 버튼 (72pt)
├─────────────────────────────────────┤
│  홈   |   일정   |   알림   |   내정보 │ ← 탭 바 (56pt)
└─────────────────────────────────────┘
```

#### 레이아웃 규칙

- **상단 네비게이션**: 
  - 높이 56pt (iOS), 56dp (Android)
  - 뒤로가기 버튼: 좌측 16pt
  - 제목: 중앙 정렬, 18pt 볼드
  - 액션 버튼: 우측 16pt

- **메인 콘텐츠**:
  - 좌우 패딩: 16pt
  - 섹션 간 간격: 24pt
  - 카드 간 간격: 12pt

- **하단 고정 버튼**:
  - 높이: 56pt + 16pt 패딩
  - 좌우 마진: 16pt
  - 그림자: 0 -2px 8px rgba(0,0,0,0.1)

- **탭 바**:
  - 높이: 56pt + Safe Area
  - 아이콘 크기: 24x24pt
  - 레이블: 12pt

### 4.2 주요 화면별 와이어프레임

#### S-003: 로그인 화면

**레이아웃**:
```
┌─────────────────────────────────────┐
│                                     │
│            WeTee Logo               │ ← 로고 (120x120pt, 중앙)
│         과외의 모든 것, 하나로         │ ← 서브 타이틀 (14pt)
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 📧  이메일                    │   │ ← 이메일 입력 (48pt 높이)
│  └─────────────────────────────┘   │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 🔒  비밀번호                  │   │ ← 비밀번호 입력 (48pt)
│  └─────────────────────────────┘   │
│                                     │
│  [     로그인     ]                 │ ← 주요 버튼 (56pt)
│                                     │
│  비밀번호를 잊으셨나요?              │ ← 링크 (14pt)
│                                     │
│  ─────── 또는 ───────              │ ← 구분선
│                                     │
│  [ 구글로 계속하기 ]                │ ← 소셜 로그인 (48pt)
│  [ 카카오로 계속하기 ]              │
│                                     │
│  아직 계정이 없으신가요? 회원가입     │ ← 링크
└─────────────────────────────────────┘
```

**주요 컴포넌트**:
- 로고: 브랜드 아이덴티티, 앱 설치 후 첫 인상
- 입력 필드: 이메일 형식 검증, 비밀번호 표시/숨김 토글
- 주요 버튼: 파란색, 비활성화 시 회색
- 소셜 로그인: 선택적 (구글, 카카오)
- 링크: 파란색, 밑줄 없음

**인터랙션**:
- 이메일 입력 → 자동 소문자 변환, @ 도메인 자동완성
- 비밀번호 입력 → 눈 아이콘 탭하면 표시/숨김 전환
- 로그인 버튼 → 로딩 인디케이터 → 성공 시 S-012 홈, 실패 시 에러 메시지
- 비밀번호 찾기 → S-005로 이동
- 회원가입 링크 → S-004로 이동

**에러 처리**:
- 이메일 형식 오류: "올바른 이메일 형식을 입력해주세요" (빨간색, 입력 필드 아래)
- 로그인 실패: "이메일 또는 비밀번호가 일치하지 않습니다" (빨간색 배너, 상단)
- 네트워크 오류: "네트워크 연결을 확인해주세요" + "다시 시도" 버튼

**참조**: F-001_회원가입_및_로그인.md 섹션 6 UI 요구사항

│  홈   |   일정   |   알림   |   내정보 │ ← 탭 바
└─────────────────────────────────────┘
```

**주요 컴포넌트**:
- **헤더**:
  - 로고: 좌측 상단, 탭 시 홈으로 (이미 홈이면 스크롤 탑)
  - 월 선택: 중앙, 탭하면 월 선택 피커 팝업
  - [+] 버튼: 우측, 탭하면 "정규 수업 추가 / 보강 예약" 바텀시트
  - 📋 버튼: 우측, 탭하면 그룹 선택 필터 바텀시트

- **뷰 전환 토글**:
  - 달력 뷰: 월간 달력 + 선택한 날짜의 일정 카드
  - 리스트 뷰: 일정 카드만 스크롤 (날짜별 그룹핑)

- **달력**:
  - 오늘: 파란색 원
  - 일정 있는 날: 하단에 작은 점 (•) 표시 (최대 3개)
  - 선택한 날: 연한 파란색 배경
  - 날짜 탭 → 해당 날짜 일정 카드로 스크롤

- **수업 카드**:
  - 시간: 14pt 볼드
  - 과목: 16pt 볼드, 배지 (파란색 배경)
  - 학생명: 14pt
  - 출결 상태: "출결 미체크" (회색), "출석" (초록색), "결석" (빨간색)
  - 액션 버튼: "출결 체크하기" (파란색), "수업 기록 작성" (초록색)
  - 카드 탭 → S-014 일정 상세

**인터랙션**:
- 월 선택 → 월 피커 팝업 (1~12월, 최대 ±1년)
- [+] 버튼 → 바텀시트 (정규 수업 추가 S-013 / 보강 예약 S-017)
- 📋 버튼 → 그룹 필터 바텀시트 (전체 / 그룹1 / 그룹2)
- 달력 날짜 탭 → 해당 날짜로 스크롤
- 수업 카드 탭 → S-014 일정 상세
- 출결 체크하기 → S-019 출결 체크 (바텀시트)
- 수업 기록 작성 → S-022 수업 기록 작성

- **선택 입력**:
  - 진도: 교재 선택 + 페이지 입력 (다중 교재 가능)
  - 학생 상태: 중간 텍스트 에리어

  - 임시 저장: 회색, 좌측 (로컬 저장, 서버 동기화 X)
  - 저장 및 공유: 파란색, 우측, 크게 (서버 저장 + 학생/학부모 알림)
  - 필수 항목 미입력 시 비활성화

**인터랙션**:

- 교재 선택 → 드롭다운 (+ 새 교재 등록 옵션)
- 페이지 입력 → 자동 계산 (67 - 53 = 14페이지)
- 다른 교재 추가 → 새 진도 입력 필드 추가 (최대 5개)
- 임시 저장 → 로컬 저장 + "임시 저장되었습니다" 토스트
- 저장 및 공유 → 서버 저장 + S-023 수업 기록 상세 + 알림 발송
- 뒤로가기 (←) → "작성 중인 내용이 있어요. 임시 저장할까요?" 다이얼로그

| Elevated | 강조 카드 (그림자) | `variant="elevated"` |
| Outlined | 테두리 카드 | `variant="outlined"` |
| Clickable | 탭 가능 카드 | `onPress={...}` |

**사용 예시**:
```jsx
<Card 
  variant="elevated" 
  onPress={() => navigate('LessonDetail')}
>
  <Text>수업 카드 내용</Text>
</Card>
```

## 7. 인터랙션 가이드라인

### 7.1 터치 타겟

#### 최소 크기

- **Android**: 48x48dp (Material Design)
- **버튼 간 간격**: 최소 8pt

- **스위치/체크박스**: 즉시 상태 변경 + 애니메이션

### 7.2 애니메이션

#### 전환 애니메이션 (Transition)

| 타입 | 지속 시간 | 이징 | 사용 사례 |
|-----|----------|------|----------|
| Fade In/Out | 200ms | ease-in-out | 모달 팝업 |
| Slide | 300ms | ease-out | 화면 전환 |
| Scale | 150ms | ease-in | 버튼 탭 |
| Bounce | 400ms | ease-out | 성공 메시지 |

#### 로딩 애니메이션

| 타입 | 사용 사례 |
|-----|----------|
| Spinner | 버튼 내 로딩 (1~3초) |
| Skeleton | 콘텐츠 로딩 (3~5초) |
| Progress Bar | 파일 업로드 |

#### 마이크로 인터랙션

- **스와이프 제스처**: 카드 삭제 (좌), 액션 (우)
- **당겨서 새로고침**: 리스트 상단에서 당기기
- **무한 스크롤**: 리스트 하단 도달 시 자동 로드

---

### 7.3 에러 처리

#### 에러 메시지 표시

| 타입 | 위치 | 지속 시간 |
|-----|------|----------|
| 인라인 | 입력 필드 아래 | 지속 |
| 토스트 | 화면 상단 | 3~4초 |
| 다이얼로그 | 화면 중앙 | 수동 닫기 |
| 빈 상태 | 콘텐츠 영역 | 지속 |

#### 에러 메시지 원칙

1. **명확성**: "오류가 발생했습니다" (X) → "이메일 형식이 올바르지 않습니다" (O)
2. **해결 방법 제시**: "다시 시도해주세요" 버튼 제공
3. **친절한 톤**: "죄송합니다" 사용하지 않기 (과도한 사과 X)

---

### 7.4 로딩 상태

#### 로딩 우선순위

1. **즉시 표시** (< 100ms): 로딩 인디케이터 생략, 즉시 결과
2. **빠른 로딩** (100ms ~ 1s): Spinner만
3. **보통 로딩** (1s ~ 3s): Skeleton UI
4. **느린 로딩** (3s ~): Progress Bar + 예상 시간 표시

#### 낙관적 업데이트 (Optimistic Update)

- **출결 체크**: 버튼 탭 즉시 UI 업데이트 → 서버 저장 (백그라운드)
- **좋아요, 체크박스**: 즉시 반영 → 실패 시 롤백
- **수업 기록 임시 저장**: 즉시 로컬 저장 → 3분마다 서버 동기화

---

### 7.5 빈 상태 (Empty State)

#### 빈 상태 표시

```
┌─────────────────────────┐
│                         │
│        📭 (아이콘)       │
│                         │
│    아직 일정이 없어요    │
│  첫 수업을 등록해보세요!  │
│                         │
│   [정규 수업 추가]       │
│                         │
└─────────────────────────┘
```

#### 빈 상태 원칙

1. **명확한 이유**: "일정이 없습니다" (X) → "아직 일정이 없어요" (O)
2. **행동 유도**: 다음 액션 버튼 제공
3. **긍정적 톤**: "등록된 항목이 없습니다" (X) → "첫 수업을 등록해보세요!" (O)
