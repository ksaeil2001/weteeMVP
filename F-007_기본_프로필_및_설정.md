## 1. 이 기능은 왜 필요한가?

### 한 문장 요약
> 사용자가 자신의 정보를 관리하고, 알림과 보안을 개인화하며, 앱 사용 경험을 자신에게 맞게 조정할 수 있게 하여 플랫폼 신뢰도와 사용 만족도를 높입니다.

### 문제 정의서와의 연결

**해결하는 문제**: 문제 1 - 파편화된 도구 사용으로 인한 데이터 관리의 혼란

문제 정의서에서 사용자들은 "여러 도구를 사용하면서 정보가 흩어지고 관리가 복잡하다"고 토로했습니다. 프로필 및 설정 기능은 사용자가 **한 곳에서 자신의 모든 정보를 관리**할 수 있게 하여, 플랫폼을 "주 도구"로 사용하도록 유도하는 핵심 기능입니다.

또한 문제 정의서의 성공 지표 중 "사용자의 90% 이상이 우리 플랫폼을 과외 관리의 주 도구로 사용"을 달성하려면, 사용자가 **자신의 필요에 맞게 앱을 커스터마이징**할 수 있어야 합니다. 알림 설정, 프로필 관리, 보안 설정은 모두 "내 앱"이라는 소유감을 만들어내는 요소입니다.

### 왜 다른 도구로 안 되는가?

**현재 사용자들이 쓰는 방법**:
- 방법 1: 각 도구마다 따로 프로필 관리 (구글, 카톡, 엑셀) → 한계: 정보가 동기화되지 않음, 일관성 없음
- 방법 2: 메모장이나 수첩에 정보 기록 → 한계: 분실 위험, 업데이트 어려움
- 방법 3: 알림을 직접 설정 못하고 모든 알림 받음 → 한계: 알림 피로도 증가, 중요한 알림 놓침

**경쟁 제품의 한계**:
- **수업기록노트**: 선생님 개인용이라 프로필 관리 간소화, 학생/학부모 연동 정보 없음
- **과릿**: 기본적인 프로필만 지원, 세밀한 알림 설정 부족
- **일반 교육 앱**: 너무 복잡한 설정 메뉴, 과외에 필요 없는 기능 과다

### 우리의 접근 방식

이 기능은 **필수 정보만 관리**하되, 사용자가 **실제로 자주 바꾸는 설정**에 집중합니다. 선생님은 수업료나 수업 시간 같은 정보를 자주 업데이트하고, 학부모는 알림 설정을 중요하게 생각합니다. 학생은 프로필 사진이나 닉네임을 바꾸고 싶어합니다. 이 모든 니즈를 **3탭 이내로 접근 가능**하게 설계하여, 복잡하지 않으면서도 필요한 모든 것을 제공합니다.

## 2. 사용자 시나리오

### 시나리오 1: 선생님이 프로필 정보 수정 (프로필 사진 + 전화번호)

**배경**: 
김선생님이 새 학생을 받으면서 학부모에게 더 전문적인 인상을 주고 싶어 프로필 사진을 업데이트하려고 합니다. 또한 최근 전화번호가 바뀌어서 연락처 정보도 수정해야 합니다.

**전제 조건**:
- 김선생님은 이미 로그인되어 있다
- 김선생님의 현재 프로필에는 기본 아이콘과 이전 전화번호가 있다

**단계**:

1. 김선생님이 앱 메인 화면에서 우측 상단의 프로필 아이콘을 탭합니다
   - 드롭다운 메뉴가 나타납니다: "내 프로필", "설정", "로그아웃"
   - 메뉴에 현재 이름과 작은 프로필 사진이 표시됩니다

2. 김선생님이 "내 프로필"을 탭합니다
   - 프로필 화면이 열립니다
   - 상단에 큰 프로필 사진 (현재는 기본 아이콘)
   - 그 아래 정보 카드: 이름, 전화번호, 이메일, 역할(선생님)
   - 하단에 "프로필 수정" 버튼

3. 김선생님이 "프로필 수정" 버튼을 탭합니다
   - 수정 모드로 전환됩니다
   - 프로필 사진 위에 카메라 아이콘과 "변경" 텍스트가 표시됩니다
   - 입력 필드들이 활성화됩니다 (이름, 전화번호는 수정 가능, 이메일은 읽기 전용)

4. 김선생님이 프로필 사진 영역을 탭합니다
   - 옵션 시트가 나타납니다: "사진 촬영", "앨범에서 선택", "취소"

5. 김선생님이 "앨범에서 선택"을 탭합니다
   - 휴대폰의 사진 앨범이 열립니다
   - 김선생님이 최근에 찍은 프로필 사진을 선택합니다

6. 김선생님이 사진을 선택하면 자동으로 프로필 화면으로 돌아옵니다
   - 새 프로필 사진이 미리보기로 표시됩니다
   - "이 사진으로 변경하시겠어요?" 확인 메시지는 없고, 바로 적용됩니다

7. 김선생님이 전화번호 입력 필드를 탭하고 새 번호를 입력합니다
   - 기존: "010-1234-5678"
   - 새로 입력: "010-9876-5432"
   - 자동으로 하이픈이 삽입됩니다

8. 김선생님이 하단의 "저장" 버튼을 탭합니다
   - 로딩 스피너가 잠깐 표시됩니다
   - 성공 메시지: "프로필이 업데이트되었습니다"
   - 프로필 화면 (읽기 모드)로 자동 전환됩니다

**결과**:
- 김선생님의 프로필 사진이 새 이미지로 변경되어, 앱 내 모든 곳(댓글, 수업 기록 등)에 표시됩니다
- 전화번호가 업데이트되어, 학부모가 연락처 확인 시 새 번호가 보입니다
- 김선생님은 더 전문적인 이미지를 학부모에게 전달할 수 있습니다

### 시나리오 2: 학부모가 알림 설정 변경 (야간 알림 끄기)

**배경**: 
박학부모는 최근 밤 늦게까지 수업 기록 알림이 와서 잠을 방해받고 있습니다. 선생님이 저녁 10시 수업 후 바로 수업 기록을 작성하면 밤 11시에 알림이 오는데, 다음 날 아침에 확인해도 되니 야간 알림을 끄고 싶습니다.

**전제 조건**:
- 박학부모는 이미 로그인되어 있다
- 현재 모든 알림이 켜져 있는 상태다

**단계**:

1. 박학부모가 앱 메인 화면에서 우측 상단의 프로필 아이콘을 탭합니다
   - 드롭다운 메뉴가 나타납니다
   - "내 프로필", "설정", "로그아웃"

2. 박학부모가 "설정"을 탭합니다
   - 설정 화면이 열립니다
   - 섹션들: "알림 설정", "보안", "앱 설정", "계정 관리"

3. 박학부모가 "알림 설정"을 탭합니다
   - 알림 설정 화면이 열립니다
   - 상단에 "알림 전체 받기" 토글 (현재 켜짐)
   - 그 아래 세부 항목들:
     - "수업 리마인더" (켜짐)
     - "수업 기록 업데이트" (켜짐)
     - "정산 알림" (켜짐)
     - "출결 변동 알림" (켜짐)
   - 하단에 "야간 알림 제한" 섹션
     - "야간에는 알림을 받지 않습니다" 토글 (현재 꺼짐)
     - 시간 선택: "오후 10:00 ~ 오전 8:00" (비활성 상태)

4. 박학부모가 "야간 알림 제한" 토글을 켭니다
   - 토글이 파란색으로 변합니다
   - 시간 선택 필드가 활성화됩니다
   - 기본값: "오후 10:00 ~ 오전 8:00"

5. 박학부모가 시간 범위를 확인합니다
   - 기본값이 적절해 보여서 그대로 둡니다
   - (만약 바꾸고 싶으면 시간 선택 필드를 탭하여 시간 피커로 조정 가능)

6. 설정 화면 상단에 "변경 사항이 자동으로 저장됩니다" 안내 문구가 표시됩니다
   - 별도의 "저장" 버튼을 누를 필요 없습니다

7. 박학부모가 뒤로가기 버튼을 탭하여 메인 화면으로 돌아갑니다
   - 토스트 메시지: "야간 알림이 제한되었습니다"

**결과**:
- 이제 오후 10시부터 오전 8시 사이에는 알림이 오지 않습니다
- 대신 아침 8시에 "밤 동안 놓친 알림 3개" 같은 요약 알림이 옵니다
- 박학부모는 방해받지 않고 잘 수 있으며, 중요한 정보는 아침에 확인할 수 있습니다

### 시나리오 3: 선생님이 비밀번호 변경 (정기 보안 강화)

**배경**: 
김선생님이 최근 보안 뉴스를 보고 정기적으로 비밀번호를 바꾸는 게 좋다는 걸 알게 되었습니다. 3개월에 한 번씩 비밀번호를 바꾸기로 결심하고 앱에서 변경하려고 합니다.

**전제 조건**:
- 김선생님은 이미 로그인되어 있다
- 현재 비밀번호: "OldPassword123!"
- 새 비밀번호: "NewSecure456@"

**단계**:

1. 김선생님이 앱 메인 화면에서 프로필 아이콘 → "설정"을 탭합니다
   - 설정 화면이 열립니다

2. 김선생님이 "보안" 섹션을 탭합니다
   - 보안 설정 화면이 열립니다
   - 항목들:
     - "비밀번호 변경"
     - "2단계 인증" (현재 꺼짐)
     - "로그인 기록"
     - "연결된 기기 관리"

3. 김선생님이 "비밀번호 변경"을 탭합니다
   - 비밀번호 변경 화면이 열립니다
   - 3개의 입력 필드:
     - "현재 비밀번호"
     - "새 비밀번호"
     - "새 비밀번호 확인"
   - 하단에 "변경하기" 버튼 (현재 비활성)

4. 김선생님이 "현재 비밀번호" 필드에 "OldPassword123!"을 입력합니다
   - 입력 시 비밀번호는 ●●●●●로 표시됩니다
   - 필드 오른쪽에 "👁" 아이콘 (탭하면 비밀번호 보임)

5. 김선생님이 "새 비밀번호" 필드에 "NewSecure456@"을 입력합니다
   - 실시간 검증이 표시됩니다:
     - ✓ 8자 이상 (초록색)
     - ✓ 대문자 포함 (초록색)
     - ✓ 숫자 포함 (초록색)
     - ✓ 특수문자 포함 (초록색)

6. 김선생님이 "새 비밀번호 확인" 필드에 "NewSecure456@"을 다시 입력합니다
   - 필드 하단에 "✓ 비밀번호가 일치합니다" (초록색)

7. 김선생님이 "변경하기" 버튼을 탭합니다 (이제 활성화됨)
   - 로딩 스피너가 표시됩니다
   - 성공 메시지: "비밀번호가 변경되었습니다"
   - 자동으로 보안 설정 화면으로 돌아갑니다

8. 추가 안내 다이얼로그가 나타납니다
   - "변경된 비밀번호로 다시 로그인해주세요"
   - "확인" 버튼

9. 김선생님이 "확인"을 탭하면 자동으로 로그아웃됩니다
   - 로그인 화면으로 이동합니다

10. 김선생님이 이메일과 새 비밀번호로 다시 로그인합니다

**결과**:
- 김선생님의 계정 보안이 강화되었습니다
- 기존 비밀번호는 더 이상 사용할 수 없습니다
- 등록된 이메일로 "비밀번호 변경 알림" 이메일이 자동 발송됩니다 (본인이 아니면 즉시 대응 가능)

### 시나리오 3-1: 선생님이 로그인 기록 조회 (보안 확인)

**배경**: 
김선생님이 최근 자신의 계정에서 의심스러운 활동이 있었는지 확인하고 싶어 로그인 기록을 조회합니다. 평소와 다른 장소나 기기에서 로그인한 기록이 있는지 점검하려고 합니다.

**전제 조건**:
- 김선생님은 이미 로그인되어 있다
- 과거에 여러 기기(스마트폰, 태블릿)에서 로그인한 기록이 있다
- 최근 7일 동안 5회 로그인 기록이 있다

**단계**:

1. 김선생님이 앱 메인 화면에서 프로필 아이콘 → "설정" → "보안"을 탭합니다
   - 보안 설정 화면이 열립니다

2. 김선생님이 "로그인 기록"을 탭합니다
   - 로그인 기록 화면이 열립니다
   - 상단 안내문: "최근 10개의 로그인 기록을 확인할 수 있습니다"

3. 화면에 로그인 기록 목록이 카드 형태로 표시됩니다
   - **각 카드 구성**:
     - 날짜/시간 (예: "2025-11-05 14:30")
     - 기기 종류 (예: "iPhone 13 Pro", "Galaxy Tab S8")
     - 위치 (예: "서울 강남구")
     - IP 주소 (일부 마스킹, 예: "121.xxx.xxx.23")
     - 로그인 방식 (예: "이메일 로그인", "카카오 로그인")
   - 현재 세션은 초록색 뱃지로 "현재 기기" 표시
   - 최신 기록이 맨 위에 표시됩니다

4. 김선생님이 목록을 스크롤하며 확인합니다
   - **정상 기록들**:
     - "2025-11-05 14:30 / iPhone 13 Pro / 서울 강남구" ✅
     - "2025-11-04 09:15 / iPhone 13 Pro / 서울 강남구" ✅
     - "2025-11-03 16:20 / Galaxy Tab S8 / 서울 강남구" ✅
   - **의심스러운 기록 발견**:
     - "2025-11-02 23:45 / Unknown Device / 부산 해운대구" ⚠️
     - IP 주소도 평소와 다름

5. 김선생님이 의심스러운 로그인 카드를 탭합니다
   - 상세 정보 바텀시트가 올라옵니다:
     ```
     로그인 상세 정보
     
     날짜/시간: 2025-11-02 23:45
     기기: Unknown Device
     OS: Android 12
     브라우저: Chrome 118
     위치: 부산 해운대구
     IP: 203.xxx.xxx.45
     로그인 방식: 이메일 로그인
     ```
   - 하단에 빨간색 버튼: "이 기기가 아닙니다"

6. 김선생님이 "이 기기가 아닙니다" 버튼을 탭합니다
   - 확인 다이얼로그가 나타납니다:
     ```
     보안 경고
     
     의심스러운 로그인으로 신고하시겠습니까?
     계정 보안을 위해 즉시 비밀번호를 변경하는 것을 권장합니다.
     
     [취소]  [신고하고 비밀번호 변경]
     ```

7. 김선생님이 "신고하고 비밀번호 변경" 버튼을 탭합니다
   - 신고가 접수되었다는 토스트 메시지: "의심스러운 로그인이 신고되었습니다"
   - 자동으로 비밀번호 변경 화면으로 이동합니다
   - 상단에 경고 배너: "⚠️ 의심스러운 로그인이 감지되었습니다. 비밀번호를 변경해주세요"

8. 김선생님이 비밀번호를 변경합니다 (시나리오 3 참조)

**결과**:
- 김선생님은 계정 보안 상태를 직접 확인할 수 있습니다
- 의심스러운 로그인 발견 시 즉시 대응 가능합니다
- 비밀번호 변경 후 모든 다른 기기에서 자동 로그아웃되어 보안이 강화됩니다
- 신고된 로그인은 관리자에게 자동 알림되어 추가 조치가 가능합니다

**예외 상황**:

| 상황 | 시스템 동작 | 사용자 메시지 |
|------|-----------|-------------|
| 로그인 기록이 없는 경우 (신규 가입자) | 빈 화면 + 안내 문구 표시 | "아직 로그인 기록이 없습니다" |
| 10개 이상의 기록이 있는 경우 | 최근 10개만 표시 | "최근 10개의 로그인 기록" |
| IP 주소를 확인할 수 없는 경우 | "확인할 수 없음"으로 표시 | (없음) |
| 위치 정보를 확인할 수 없는 경우 | "위치 정보 없음"으로 표시 | (없음) |

### 시나리오 4: 학생이 계정 탈퇴 (과외 종료 후)

**배경**: 
이학생이 대학에 합격하여 더 이상 과외를 받지 않게 되었습니다. 앱을 계속 설치해둘 필요가 없어서 계정을 탈퇴하려고 합니다.

**전제 조건**:
- 이학생은 이미 로그인되어 있다
- 이학생이 속한 과외 그룹이 있다
- 미결제된 수업료가 없다

**단계**:

1. 이학생이 앱 메인 화면에서 프로필 아이콘 → "설정"을 탭합니다

2. 이학생이 설정 화면을 아래로 스크롤하여 "계정 관리" 섹션을 찾습니다
   - "로그아웃"
   - "계정 탈퇴" (빨간색 텍스트)

3. 이학생이 "계정 탈퇴"를 탭합니다
   - 경고 화면이 나타납니다
   - 상단에 경고 아이콘 (⚠️)
   - 제목: "정말 탈퇴하시겠어요?"
   - 설명:
     - "탈퇴하면 다음 정보가 영구 삭제됩니다:"
     - "• 프로필 정보"
     - "• 수업 기록 및 진도"
     - "• 과외 그룹 연결"
     - "• 출결 기록"
     - 
     - "단, 정산 관련 데이터는 법적 의무에 따라 5년간 보관됩니다"
   - 체크박스: "위 내용을 확인했으며 탈퇴에 동의합니다"
   - 버튼: "취소" (회색), "탈퇴하기" (빨간색, 현재 비활성)

4. 이학생이 체크박스를 체크합니다
   - "탈퇴하기" 버튼이 활성화됩니다

5. 이학생이 "탈퇴하기" 버튼을 탭합니다
   - 최종 확인 다이얼로그가 나타납니다
   - "비밀번호를 입력하여 본인 확인을 해주세요"
   - 비밀번호 입력 필드
   - 버튼: "취소", "확인"

6. 이학생이 비밀번호를 입력하고 "확인"을 탭합니다
   - 로딩 스피너가 표시됩니다
   - 탈퇴 처리 중 (3-5초)

7. 탈퇴 완료 화면이 나타납니다
   - "탈퇴가 완료되었습니다"
   - "그동안 이용해주셔서 감사합니다"
   - 등록된 이메일로 탈퇴 확인 이메일이 발송됩니다
   - "앱 종료" 버튼

8. 이학생이 "앱 종료"를 탭하면 앱이 완전히 종료됩니다

**결과**:
- 이학생의 계정이 삭제되고 더 이상 로그인할 수 없습니다
- 선생님과 학부모에게 "이학생님이 그룹을 떠났습니다" 알림이 전송됩니다
- 해당 그룹의 학생 목록에서 이학생이 제거됩니다
- 수업 기록 등 과거 데이터는 선생님 쪽에만 보관됩니다 (법적 의무)

### 시나리오 5: 학부모가 앱 언어 설정 변경 (한국어 → 영어)

**배경**: 
박학부모는 한국 거주 외국인이며, 한국어를 잘 못합니다. 처음에는 자녀가 앱 사용을 도와줘서 한국어로 설정했지만, 이제 직접 사용하려니 영어가 필요합니다.

**전제 조건**:
- 박학부모는 이미 로그인되어 있다
- 현재 언어: 한국어

**단계**:

1. 박학부모가 프로필 아이콘 → "설정"을 탭합니다

2. 박학부모가 "앱 설정" 섹션을 탭합니다
   - 앱 설정 화면이 열립니다
   - 항목들:
     - "언어" (현재: 한국어)
     - "테마" (현재: 자동)
     - "기본 화면" (현재: 홈)

3. 박학부모가 "언어"를 탭합니다
   - 언어 선택 화면이 나타납니다
   - 목록:
     - 한국어 (✓ 선택됨)
     - English
     - 日本語
   - 상단 안내: "Language / 言語"

4. 박학부모가 "English"를 탭합니다
   - 확인 다이얼로그: "Change language to English?" / "언어를 영어로 변경하시겠습니까?"
   - 버튼: "Cancel / 취소", "Change / 변경"

5. 박학부모가 "Change / 변경"을 탭합니다
   - 로딩 스피너 (1-2초)
   - 앱 전체 텍스트가 영어로 변경됩니다
   - 토스트 메시지: "Language changed to English"

6. 박학부모가 메인 화면으로 돌아갑니다
   - 모든 메뉴, 버튼, 안내 문구가 영어로 표시됩니다

**결과**:
- 박학부모는 이제 앱을 영어로 편하게 사용할 수 있습니다
- 알림 메시지도 영어로 발송됩니다
- 선생님이 작성한 수업 기록은 원문 그대로 표시됩니다 (번역되지 않음)

| 지원하지 않는 언어 선택 시도 | (발생 불가) | 선택 목록에 지원 언어만 표시 | 예방 설계 |
| 다크 모드가 OS에서 지원 안 됨 | "이 기기는 다크 모드를 지원하지 않아요" | 테마 옵션에서 "다크" 제거, "라이트"만 선택 가능 | 호환성 |

## 4. 비즈니스 규칙

### 프로필 정보 규칙

- **프로필 사진**:
  - 허용 형식: JPG, PNG, HEIC
  - 최대 파일 크기: 10MB
  - 권장 해상도: 500×500px 이상
  - 사진이 없으면 기본 아이콘(이니셜 기반) 자동 생성

- **이름**:
  - 최소 1자, 최대 50자
  - 한글, 영어, 숫자, 일부 특수문자(. -) 허용
  - 이모지 사용 불가
  - 필수 항목

- **전화번호**:
  - 형식: 010-XXXX-XXXX (자동으로 하이픈 삽입)
  - 최소 10자리, 최대 11자리
  - 숫자만 입력 가능
  - 선택 항목 (입력하지 않아도 됨)

- **이메일**:
  - 읽기 전용 (F-001에서 설정한 이메일은 수정 불가)
  - 변경하려면 고객센터 문의 필요
  - 이유: 이메일은 계정의 고유 식별자

### 알림 설정 규칙

- **알림 항목**:
  - "수업 리마인더": 수업 1시간 전 알림 (변경 불가)
  - "수업 기록 업데이트": 선생님이 수업 기록 작성 시 즉시 알림
  - "정산 알림": 결제 요청, 결제 완료, 결제 실패 알림 (끌 수 없음)
  - "출결 변동 알림": 출결 수정 시 알림
  - 각 항목별로 개별 On/Off 가능

- **야간 알림 제한**:
  - 시간 범위 설정 가능 (예: 오후 10:00 ~ 오전 8:00)
  - 제한 시간 동안 알림 미발송
  - 대신 제한 종료 직후(예: 오전 8:00) "밤 동안 놓친 알림 X개" 요약 알림 발송
  - 단, 긴급 알림(예: 정산 실패)은 제한 무시하고 즉시 발송

- **푸시 알림 권한**:
  - 최초 가입 시 푸시 알림 권한 요청
  - 사용자가 거부하면 앱 내 알림(인앱 배너)으로만 표시
  - 설정 화면에서 "휴대폰 설정으로 이동" 버튼 제공

### 보안 설정 규칙

- **비밀번호 변경**:
  - 현재 비밀번호 확인 필수
  - 새 비밀번호 규칙: F-001 참조 (8자 이상, 대문자/숫자/특수문자 포함)
  - 현재 비밀번호와 동일한 비밀번호 사용 불가
  - 최근 3개월 내 사용한 비밀번호 재사용 불가
  - 변경 후 자동 로그아웃 (다시 로그인 필요)
  - 변경 완료 시 등록된 이메일로 알림 발송

- **로그인 기록**:
  - 최근 10개 로그인 기록 표시
  - **표시 정보**:
    - 날짜/시간 (예: "2025-11-05 14:30")
    - 기기 종류 (예: "iPhone 13 Pro", "Galaxy Tab S8")
    - 운영체제 (예: "iOS 17.1", "Android 13")
    - 위치 (도시 단위, 예: "서울 강남구", "부산 해운대구")
    - IP 주소 (일부 마스킹, 예: "121.xxx.xxx.23")
    - 로그인 방식 (이메일, 카카오, 구글 등)
  - **보안 기능**:
    - 현재 세션은 "현재 기기" 뱃지로 구분
    - 의심스러운 로그인 발견 시 "이 기기가 아닙니다" 신고 가능
    - 신고 후 즉시 비밀번호 변경 유도
    - 비밀번호 변경 시 신고된 기기 자동 로그아웃
  - **자동 보안 경고**:
    - 새로운 기기에서 로그인 시 이메일로 알림 발송
    - 다른 나라에서 로그인 시 추가 인증 요구
    - 짧은 시간 내 여러 위치 로그인 시 자동 차단 고려

- **연결된 기기 관리**:
  - 현재 로그인된 모든 기기 목록 표시
  - 각 기기: 기기명, 마지막 접속 시간
  - 선택적으로 특정 기기 강제 로그아웃 가능
  - "모든 기기 로그아웃" 버튼 (비밀번호 확인 필요)

### 계정 탈퇴 규칙

- **탈퇴 가능 조건**:
  - 미결제 수업료가 없어야 함
  - 선생님의 경우: 활성 그룹이 없거나 다른 선생님에게 인계 완료
  - 학생/학부모의 경우: 언제든 탈퇴 가능

- **탈퇴 시 삭제되는 데이터**:
  - 프로필 정보 (이름, 사진, 전화번호)
  - 알림 설정
  - 로그인 기록
  - 수업 기록 (학생/학부모 쪽만 삭제, 선생님 쪽은 보관)
  - 출결 기록 (학생 쪽만 삭제, 선생님 쪽은 보관)

- **탈퇴 후 보관되는 데이터 (법적 의무)**:
  - 정산 내역: 5년간 보관 (전자상거래법)
  - 결제 정보: 5년간 보관
  - 이메일 주소: 30일간 보관 (재가입 방지)

- **탈퇴 절차**:
  1. 체크박스 동의 필수
  2. 비밀번호 확인 필수 (본인 인증)
  3. 최종 확인 다이얼로그
  4. 탈퇴 완료 시 이메일 발송

- **재가입 제한**:
  - 탈퇴 후 30일간 같은 이메일로 재가입 불가
  - 이유: 악용 방지 (탈퇴 → 재가입 반복)

### 앱 설정 규칙

- **언어**:
  - 지원 언어: 한국어, English, 日本語
  - 변경 즉시 적용 (앱 재시작 불필요)
  - 수업 기록 등 사용자 입력 콘텐츠는 번역하지 않음

- **테마**:
  - 옵션: 라이트, 다크, 자동 (시스템 설정 따름)
  - 변경 즉시 적용

- **기본 화면**:
  - 앱 실행 시 처음 보이는 화면 선택
  - 옵션: 홈, 일정, 진도, 정산
  - 사용자 역할에 따라 권장 화면 다름 (선생님: 일정, 학부모: 진도)

### 프로필 화면

**레이아웃**:
- 상단: 큰 원형 프로필 사진 (중앙 정렬)
- 중앙: 정보 카드 (이름, 전화번호, 이메일, 역할)

**주요 컴포넌트**:
- **프로필 사진**:
  - 크기: 120×120px
  - 원형

  - 수정 모드에서는 사진 위에 반투명 오버레이 + 카메라 아이콘

- **정보 카드**:
  - 흰색 배경 (라이트 모드) / 진한 회색 (다크 모드)
  - 둥근 모서리
  - 각 항목: 레이블 (작은 회색 글씨) + 값 (큰 검은 글씨)
  - 예: "이름" (레이블) / "김선생" (값)

- **버튼**:
  - "프로필 수정": 파란색, 전체 너비
  - 수정 모드에서는 "저장" (파란색), "취소" (회색)

**인터랙션**:
- 프로필 사진 탭 → 사진 변경 옵션 시트
- 수정 모드에서 입력 필드 활성화
- 실시간 검증 (전화번호 형식, 이름 길이 등)

- **경고 메시지**:
  - 큰 빨간색 아이콘
  - 제목: 볼드, 검은색
  - 삭제될 항목 목록 (불릿 포인트)
  - 보관될 항목 안내 (작은 회색 글씨)

- **체크박스**:
  - "위 내용을 확인했으며 탈퇴에 동의합니다"
  - 필수 체크

- **버튼들**:
  - "취소": 회색, 왼쪽
  - "탈퇴하기": 빨간색, 오른쪽
  - 체크박스 선택 시에만 "탈퇴하기" 활성화

**인터랙션**:
- "탈퇴하기" 탭 → 비밀번호 확인 다이얼로그
- 비밀번호 맞으면 → 탈퇴 처리
- 완료 화면 → "앱 종료" 버튼

## 7. 다른 기능과의 연결

### 의존하는 기능 (이것들이 먼저 있어야 함)

- **F-001: 회원가입 및 로그인** - 이유: 프로필은 가입한 사용자에게만 존재. 비밀번호 변경 규칙은 F-001과 동일

- **F-002: 과외 그룹 생성 및 매칭** - 이유: 계정 탈퇴 시 활성 그룹 확인 필요. 그룹이 있으면 탈퇴 불가

- **F-006: 과외비 자동 정산** - 이유: 계정 탈퇴 시 미결제 수업료 확인 필요. 미결제가 있으면 탈퇴 불가

### 이 기능에 의존하는 기능 (이것 다음에 만들어질 것들)

- **F-008: 필수 알림 시스템** - 이유: F-007에서 설정한 알림 설정을 F-008이 참조하여 알림 발송 여부 결정

- **F-009: 숙제 관리 (2단계)** - 이유: 숙제 관련 알림도 F-007의 알림 설정을 따름

- **F-010: 학습 대시보드 (2단계)** - 이유: 프로필 정보(이름, 사진)가 대시보드에 표시됨

### 함께 사용되는 기능 (밀접하게 연결됨)

- **F-001: 회원가입 및 로그인** - 관계: 비밀번호 변경 시 F-001의 비밀번호 규칙을 따름. 이메일 주소는 F-001에서 설정하고 F-007에서는 읽기 전용

- **모든 기능** - 관계: 프로필 사진과 이름은 모든 기능에서 공통으로 사용됨 (댓글, 수업 기록, 정산 내역 등)

## 8. 향후 고려사항 (2단계 이후)

### 기능 확장

- **2단계 인증 (2FA)**:
  - SMS 또는 이메일을 통한 추가 인증
  - 로그인 시 비밀번호 + 인증 코드 입력
  - 왜 2단계인가: MVP에서는 비밀번호만으로 충분. 보안 강화는 사용자 피드백 후

- **소셜 계정 연동**:
  - 카카오톡, 네이버, 구글 계정과 연동
  - 프로필 정보 자동 동기화
  - 왜 2단계인가: 이메일 가입만으로 MVP 검증. 소셜 로그인은 F-001의 2단계 확장

- **프로필 공개/비공개 설정**:
  - 다른 사용자에게 프로필 정보 공개 여부 선택
  - 예: 전화번호는 비공개, 이름과 사진만 공개
  - 왜 2단계인가: MVP에서는 그룹 내에서만 정보 공유. 커뮤니티 기능 추가 시 필요

- **알림 시간대 세밀 설정**:
  - 요일별로 다른 야간 알림 시간 설정
  - 예: 주중에는 오후 10시, 주말에는 오후 11시
  - 왜 2단계인가: 복잡도 증가. 일단 통일된 시간대로 충분

- **프로필 백업/복원**:
  - 계정 탈퇴 전 데이터 백업
  - 30일 내 재가입 시 데이터 복원 옵션
  - 왜 2단계인가: 기술적 복잡도. MVP는 탈퇴 시 완전 삭제

### 개선 아이디어

- **프로필 완성도 표시**:
  - 프로필 정보 입력률 시각화 (예: "70% 완성")
  - 미입력 항목 안내 (예: "전화번호를 입력하면 학부모가 쉽게 연락할 수 있어요")
  - 왜 나중인가: 게이미피케이션 요소. 핵심 기능 후 추가

- **다크 모드 스케줄링**:
  - 시간대별 자동 테마 변경 (예: 오후 7시부터 다크 모드)
  - 왜 나중인가: 사용자 요청이 있을 때 추가

- **프로필 배지 시스템**:
  - "신뢰 선생님", "성실한 학생" 같은 배지 표시
  - F-011 (학생 포트폴리오)와 연계
  - 왜 나중인가: 2단계 기능과 연계 필요

- **알림 요약 보고서**:
  - 주간/월간 알림 통계 (예: "이번 주에 알림 20개 받았어요")
  - 가장 많이 받은 알림 유형 분석
  - 왜 나중인가: 추가 데이터 수집 및 분석 필요

## 9. 성공 지표

### 정량적 지표

| 지표명 | 측정 방법 | 목표값 | 측정 시점 |
|--------|---------|--------|----------|
| 프로필 완성률 | 전체 사용자 중 프로필 사진 + 전화번호를 모두 입력한 비율 | 80% 이상 | 가입 후 1주일 |
| 알림 설정 변경률 | 전체 사용자 중 최소 1개 이상 알림 설정을 변경한 비율 | 60% 이상 | 가입 후 1개월 |
| 야간 알림 제한 사용률 | 야간 알림 제한을 활성화한 사용자 비율 | 40% 이상 | 가입 후 2주일 |
| 비밀번호 변경 빈도 | 사용자당 평균 비밀번호 변경 횟수 | 분기당 0.5회 이상 | 분기별 |
| 계정 탈퇴율 | 전체 사용자 중 탈퇴한 비율 | 10% 미만 | 6개월 |
| 설정 화면 접근 빈도 | 사용자당 월간 평균 설정 화면 방문 횟수 | 3회 이상 | 매월 |

### 정성적 지표

- **사용자 피드백**: 
  - 기대: "설정이 직관적이고 찾기 쉬워요", "알림을 내가 원하는 대로 조정할 수 있어서 좋아요"
  
- **문제 해결도**: 
  - 문제 정의서의 "파편화된 도구 사용" 해결 기여
  - 목표: "프로필 관리를 한 곳에서 할 수 있어서 편하다" 80% 이상

- **신뢰도 향상**:
  - "내 정보를 안전하게 관리할 수 있다는 믿음" 85% 이상 (5점 만점 중 4점 이상)
  - 비밀번호 변경 기능이 신뢰도에 기여

## 10. 알려진 제약사항 및 트레이드오프

### 기술적 제약

**이메일 주소 수정 불가**:
- **제약사항**: 가입 시 설정한 이메일은 수정할 수 없음
- **이유**: 
  - 이메일은 계정의 고유 식별자 (Primary Key 역할)
  - 이메일 변경을 허용하면 중복 가입, 계정 양도 등 악용 가능성
- **대응**: 이메일 변경이 꼭 필요하면 고객센터 문의 (수동 처리)
- **트레이드오프**: 사용자 편의성 < 계정 보안 및 무결성

**프로필 사진 크기 제한 (10MB)**:
- **제약사항**: 10MB 이상의 이미지는 업로드 불가
- **이유**: 서버 저장 용량 및 네트워크 비용 절감
- **실제 영향**: 대부분의 스마트폰 사진은 3-5MB. 10MB는 충분히 큰 크기
- **완화책**: 업로드 전 자동 압축 (품질 유지하면서 용량 감소)

**탈퇴 후 30일간 재가입 불가**:
- **제약사항**: 탈퇴 후 30일 동안 같은 이메일로 재가입 불가
- **이유**: 
  - 탈퇴 → 재가입 반복으로 정산 회피 시도 방지
  - 법적 의무 보관 기간 확보 (데이터 완전 삭제까지 시간 필요)
- **트레이드오프**: 사용자 편의성 < 악용 방지 및 법적 준수

### 의도적 선택

**정산 알림은 끌 수 없음**:
- **포기한 유연성**: 사용자가 원하는 대로 모든 알림을 끌 수 있는 자유
- **얻은 것**: 
  - 금전 거래 분쟁 방지 (사용자가 알림을 놓쳐서 결제 지연될 위험 차단)
  - 플랫폼 신뢰도 향상 (정산은 투명하게 관리)
- **이유**: 정산은 과외의 핵심. 알림을 끄면 사용자 스스로에게 불이익

**학생/학부모는 로그인 기록 조회 불가 (MVP)**:
- **포기한 기능**: 모든 사용자에게 동일한 보안 기능 제공
- **얻은 것**: 
  - 개발 리소스 절약 (우선순위가 높은 선생님만 지원)
  - 선생님의 보안 요구 수준이 더 높음 (금전 관리 주체)
- **이유**: MVP에서는 선생님 중심 기능에 집중. 학생/학부모는 2단계에서 추가

**비밀번호 변경 후 자동 로그아웃**:
- **포기한 편의성**: 변경 후 계속 사용 가능
- **얻은 것**: 
  - 보안 강화 (다른 기기에서 로그인된 세션 즉시 종료)
  - 비밀번호 변경의 중요성 강조
- **이유**: 보안 베스트 프랙티스. 변경 후 재로그인은 업계 표준

**프로필 수정 시 별도의 "수정" 버튼 필요**:
- **포기한 편의성**: 항상 수정 가능하게 필드를 열어두기
- **얻은 것**: 
  - 실수로 정보 변경 방지
  - 읽기 모드와 수정 모드 명확히 구분
- **이유**: 중요한 정보(이름, 전화번호)는 신중하게 변경해야 함

**언어는 3개만 지원 (한국어, 영어, 일본어)**:
- **포기한 기능**: 다국어 완전 지원 (중국어, 스페인어 등)
- **얻은 것**: 
  - 번역 및 유지보수 비용 절감
  - MVP 빠른 출시
- **이유**: 목표 시장은 한국. 외국인 사용자는 주로 영어권 또는 일본어권
- **향후 계획**: 사용자 요청에 따라 언어 추가

**탈퇴 시 일부 데이터는 영구 보관**:
- **포기한 것**: 완전한 "잊혀질 권리" 보장
- **얻은 것**: 
  - 법적 의무 준수 (전자상거래법)
  - 정산 분쟁 발생 시 증거 자료 확보
- **이유**: 과외비 거래 기록은 5년간 보관 의무. 이는 법적 요구사항
