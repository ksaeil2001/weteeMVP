## 1. 이 기능은 왜 필요한가?

### 한 문장 요약
> 선생님이 매 수업 후 오늘 배운 내용과 학생 상태를 간편하게 기록하고, 이를 학부모에게 자동으로 공유하여 선생님의 노력이 투명하게 전달되게 하며, 교재별 진도를 체계적으로 추적하여 다음 수업 준비를 효율화합니다.

### 문제 정의서와의 연결

**해결하는 문제**: 문제 3 - 학부모와의 진도 공유 어려움

문제 정의서에서:
> "수업 진도나 학습 내용을 엑셀로 진도표 만들어 활용. 노션으로 만들어보려다가 포기"

> "공유하지 않고 상담 시 보내드림"

> "학부모가 과외 교사를 불신하는 주된 이유는 교사의 '무관심한 태도'(36.4%)"

> "실제로는 선생님이 무관심한 게 아니라, 그 관심과 노력이 학부모에게 제대로 전달되지 않기 때문"

**핵심 문제**:
- 선생님은 열심히 가르치는데, 학부모는 "우리 아이가 뭘 배우는지" 모름
- 수업 후 진도를 기록하려면 엑셀을 열고, 노션을 열고, 문서를 찾고... → 귀찮아서 안 함
- 기록했어도 학부모에게 공유하려면 캡처해서 카톡으로 보내야 함 → 더 귀찮음
- 결과: 선생님의 노력이 보이지 않음 → 학부모는 "무관심한 선생님"으로 오해 → 신뢰 깨짐

**실제 사례**:
- 리서치에서 "진도 공유 필요성"이 초기 예상(중간)보다 **실제 사용자 요구가 높음(높음)**으로 상향됨
- "정기적인 학업 보고 및 피드백 시스템을 디지털로 구축할 경우, 학부모 참여도가 25% 증가하고 학생들의 책임감이 향상"

### 왜 다른 도구로 안 되는가?

**현재 사용자들이 쓰는 방법**:
- **엑셀 진도표**: 
  - 한계: 작성이 번거로움. 학부모가 볼 수 없음. 파일을 찾아서 열어야 함. 모바일에서 수정 어려움
  
- **노션 진도 페이지**: 
  - 한계: "만들어보려다가 포기" - 설정이 복잡함. 학부모에게 권한 공유 어려움. 과외 전용이 아니라 불필요한 기능 많음
  
- **카카오톡 메시지**:
  - 한계: "오늘 2차함수 미분 배웠어요" 같은 짧은 메시지만 가능. 체계적 누적 불가. 나중에 찾기 어려움
  
- **구글 닥스 공유**:
  - 한계: 매번 문서 링크 복사해서 보내야 함. 학부모가 볼지 안 볼지 확인 불가. 교재별 정리 어려움

**경쟁 제품의 한계**:
- **과릿**: 수업 리포트 실시간 공유가 강점이나, 정산 기능 부재. 우리는 수업 기록 → 출결 → 정산이 자동 연결
- **수업기록노트**: 선생님 개인용 기록. 학부모와 실시간 공유 안 됨. "캡쳐나 메시지 전송을 별도로 해야 하는 한계"
- **김과외**: 플랫폼 내 수업 내용 관리는 있으나, 자체 매칭만 지원. 외부 과외에는 사용 불가

### 우리의 접근 방식

1. **수업 직후 즉시 기록**: 수업 끝나고 2-3분 안에 간단한 폼 작성. "오늘 뭐 했지?" 고민할 필요 없도록 구조화된 입력 항목 제공

2. **자동 공유**: 선생님이 기록 완료 버튼 누르면 즉시 학생과 학부모에게 알림 전송. 별도로 카톡 보낼 필요 없음

3. **교재 중심 진도 추적**: "자이스토리 수학 2 - 53페이지까지" 식으로 교재별 진도 자동 계산. 다음 수업 시작 전 "지난번 어디까지 했지?" 바로 확인 가능

4. **누적 히스토리**: 모든 수업 기록이 시간순 정렬. 한 달 전 내용도 쉽게 찾기. 학기말 총정리 시 유용

5. **템플릿 지원**: 매번 같은 형식으로 쓰는 선생님을 위해 "지난번 기록 복사" 기능. 바뀐 부분만 수정

## 2. 사용자 시나리오

### 시나리오 1: 선생님이 수업 후 첫 수업 기록을 작성하는 경우

**배경**: 
김선생님이 이학생과 2시간 수학 수업을 마쳤습니다. 오늘은 이차방정식의 판별식을 배웠고, 학생의 이해도가 좋았습니다. 이제 수업 기록을 작성하려고 합니다.

**전제 조건**:
- 김선생님은 이미 로그인되어 있음
- 오늘 일정에 "이학생 수학 과외" 일정이 있고, 출결은 "출석"으로 체크됨 (F-004 완료)
- 아직 이 일정에 대한 수업 기록이 없음

**단계**:

1. 김선생님이 앱 메인 화면의 "오늘의 수업" 섹션을 확인합니다.
   - "이학생 수학 과외" 카드가 보입니다
   - 카드에 출석 체크는 되어있지만, "수업 기록 미작성" 뱃지가 표시됩니다
   - 카드 하단에 **"수업 기록 작성"** 버튼이 있습니다

2. **"수업 기록 작성"** 버튼을 누릅니다.
   - **"수업 기록 작성"** 화면으로 전환됩니다

3. 수업 기록 폼이 나타납니다:
   
   **상단 정보 (자동 표시)**:
   - 학생 이름: "이학생"
   - 과목: "수학"
   - 수업 날짜: "2025년 11월 4일 (월)"
   - 수업 시간: "오후 3:00 - 5:00 (2시간)"
   
   **입력 섹션 1: 오늘 배운 내용** (필수)
   - 큰 텍스트 입력창
   - 플레이스홀더: "예) 이차방정식의 판별식 개념 설명 및 문제 풀이"
   - 안내: "학부모님께 공유되는 내용입니다"
   
   **입력 섹션 2: 진도 (선택)**
   - 제목: "오늘 진행한 진도"
   - **첫 번째 교재 진도**:
     - 교재 선택 드롭다운: "자이스토리 수학 II" (이전에 등록한 교재 목록)
     - 진도 입력: "53페이지" ~ "67페이지"
   - **+ "다른 교재 진도 추가" 버튼**
     - 클릭 시 추가 교재 진도 입력 필드가 나타남
     - 한 수업에서 여러 교재를 사용한 경우 각각 기록 가능
     - 예: 수학 문제집 + 영문법 교재를 같은 수업에서 진행
   - "+ 새 교재 등록" 버튼 (드롭다운 하단)
   
   **입력 섹션 3: 학생 상태 (선택)**
   - 텍스트 입력창
   - 플레이스홀더: "예) 오늘 집중력이 좋았어요. 어려운 문제도 스스로 풀려고 노력했습니다."
   
   **입력 섹션 4: 숙제 (선택)**
   - 텍스트 입력창
   - 플레이스홀더: "예) 교과서 67~70페이지 문제 풀어오기"
   - 안내: "학생에게 알림이 전송됩니다"

4. 김선생님이 각 섹션을 채웁니다:
   - 오늘 배운 내용: "이차방정식의 판별식(b²-4ac) 개념 이해. 근의 개수 판별 연습. 응용 문제 5개 풀이"
   - 진도: 자이스토리 수학 II, 53 ~ 67페이지
   - 학생 상태: "판별식 개념을 빠르게 이해했어요. 어려운 문제도 포기하지 않고 끝까지 풀려고 했습니다. 다음 시간에 활용 문제 더 풀어보면 좋을 것 같아요."
   - 숙제: "교과서 67~70페이지 문제 풀어오기. 틀린 문제는 다시 풀어보세요."

5. 모든 필수 항목을 채우고 화면 하단의 **"저장 및 공유"** 버튼을 누릅니다.

6. 확인 팝업이 나타납니다:
   - "수업 기록을 저장하고 이학생, 이학생 학부모님께 공유할까요?"
   - "학부모님께는 푸시 알림과 앱 내 알림이 전송됩니다."
   - **"확인"**, **"취소"** 버튼

7. 김선생님이 **"확인"**을 누릅니다.
   - 성공 메시지: "수업 기록이 저장되었습니다"
   - 자동으로 이전 화면으로 돌아갑니다
   - "이학생 수학 과외" 카드에 "기록 완료" 초록색 뱃지가 표시됩니다

**결과**:
- 이학생과 이학생 학부모는 즉시 푸시 알림을 받습니다: "이학생님의 오늘 수업 기록이 등록되었습니다"
- 학부모가 알림을 탭하면 수업 기록 상세 화면으로 이동하여 내용 확인 가능
- 교재 "자이스토리 수학 II"의 진도가 자동으로 업데이트됨 (53 → 67페이지로)
- 숙제는 학생의 "내 숙제" 목록에 자동 추가됨

### 시나리오 2: 학부모가 수업 기록을 확인하는 경우

**배경**: 
이학생 학부모 박학부모는 방금 "오늘 수업 기록이 등록되었습니다" 알림을 받았습니다. 자녀가 오늘 무엇을 배웠는지 확인하려고 합니다.

**전제 조건**:
- 박학부모는 이미 로그인되어 있음
- 김선생님이 수업 기록을 작성하고 공유함 (시나리오 1 완료)

**단계**:

1. 박학부모가 푸시 알림을 탭합니다.
   - 앱이 열리고 **"수업 기록 상세"** 화면으로 바로 이동합니다

2. 수업 기록 상세 화면이 나타납니다:
   
   **상단**:
   - 날짜: "2025년 11월 4일 (월)"
   - 과목: "수학" 뱃지
   - 수업 시간: "오후 3:00 - 5:00 (2시간)"
   - 출결 상태: "출석" 초록색 뱃지
   
   **섹션 1: 오늘 배운 내용**
   - 제목: "오늘 배운 내용"
   - 내용: "이차방정식의 판별식(b²-4ac) 개념 이해. 근의 개수 판별 연습. 응용 문제 5개 풀이"
   
   **섹션 2: 진도**
   - 제목: "진도"
   - 교재: "자이스토리 수학 II"
   - 진도: "53 ~ 67페이지 (14페이지 진행)"
   - 작은 글씨로: "전체 진도: 240페이지 중 67페이지 (27.9%)"
   
   **섹션 3: 학생 상태**
   - 제목: "학생 상태"
   - 내용: "판별식 개념을 빠르게 이해했어요. 어려운 문제도 포기하지 않고 끝까지 풀려고 했습니다. 다음 시간에 활용 문제 더 풀어보면 좋을 것 같아요."
   
   **섹션 4: 숙제**
   - 제목: "숙제"
   - 내용: "교과서 67~70페이지 문제 풀어오기. 틀린 문제는 다시 풀어보세요."
   - 작은 글씨로: "이학생님에게도 알림이 전송되었습니다"

3. 박학부모가 화면을 아래로 스크롤하면 **"이전 수업 기록"** 버튼이 보입니다.
   - 버튼을 누르면 과거 수업 기록 목록으로 이동

4. 박학부모가 화면 우측 상단의 **"공유"** 아이콘을 탭합니다.
   - 공유 옵션이 나타납니다: "카카오톡으로 공유", "이미지로 저장", "링크 복사"
   - 선택하면 해당 방식으로 공유 가능

**결과**:
- 학부모는 자녀가 오늘 무엇을 배웠는지, 어떤 상태였는지, 숙제는 무엇인지 한눈에 확인
- 선생님의 세심한 관찰과 피드백을 보고 신뢰감 형성
- 필요시 다른 가족 구성원(배우자 등)에게도 쉽게 공유 가능

### 시나리오 3: 학생이 수업 기록을 보고 숙제를 확인하는 경우

**배경**: 
이학생은 오늘 수업이 끝난 후 저녁에 앱을 열어 오늘 배운 내용을 복습하고, 숙제를 확인하려고 합니다.

**전제 조건**:
- 이학생은 이미 로그인되어 있음
- 김선생님이 수업 기록을 작성하고 공유함 (시나리오 1 완료)

**단계**:

1. 이학생이 앱 메인 화면을 확인합니다.
   - **"최근 수업 기록"** 섹션에 오늘 수업이 최상단에 표시됩니다
   - "2025.11.4 (월) 수학 - 이차방정식의 판별식..."
   - 카드를 탭합니다

2. 수업 기록 상세 화면으로 이동합니다.
   - 학부모가 보는 화면과 동일한 내용이 표시됩니다
   - 오늘 배운 내용, 진도, 학생 상태, 숙제 모두 확인 가능

3. 이학생이 **"숙제"** 섹션을 확인합니다.
   - "교과서 67~70페이지 문제 풀어오기. 틀린 문제는 다시 풀어보세요."
   - 숙제 옆에 **"완료 체크"** 버튼이 있습니다 (선택 사항)

4. 이학생이 메인 화면으로 돌아가 **"내 숙제"** 탭을 선택합니다.
   - 숙제 목록이 시간순으로 정렬되어 표시됩니다
   - 오늘 받은 숙제가 최상단에 "미완료" 상태로 표시됩니다
   - 각 숙제마다 마감일(다음 수업 날짜)이 표시됩니다

5. 이학생이 숙제를 다 끝낸 후, 숙제 항목 옆의 체크박스를 탭합니다.
   - 숙제 상태가 "완료"로 변경됩니다
   - 완료 시간이 자동 기록됩니다
   - 선택 사항: "완료 인증" 버튼을 눌러 사진 첨부 가능

**결과**:
- 학생은 오늘 배운 내용을 다시 확인하며 복습 가능
- 숙제를 잊어버리지 않고 체계적으로 관리
- 선택적으로 숙제 완료를 선생님에게 알릴 수 있음

### 시나리오 4: 선생님이 진도 히스토리를 확인하는 경우

**배경**: 
김선생님은 이학생과 한 달간 수업을 진행했습니다. 이제 학기 중간 상담 전에 그동안의 진도와 수업 내용을 정리하려고 합니다.

**전제 조건**:
- 김선생님은 이미 로그인되어 있음
- 한 달간 총 8회의 수업 기록이 작성되어 있음
- 교재는 "자이스토리 수학 II"를 사용 중

**단계**:

1. 김선생님이 "이학생 수학 과외" 그룹 상세 화면으로 이동합니다.

2. **"진도" 탭**을 선택합니다.
   - 화면 상단에 교재 목록이 카드 형태로 표시됩니다
   - "자이스토리 수학 II" 카드를 탭합니다

3. 교재별 진도 상세 화면이 나타납니다:
   
   **상단 요약**:
   - 진도율 바: 67 / 240 페이지 (27.9%)
   - 시작일: 2025년 10월 1일
   - 최근 수업: 2025년 11월 4일
   - 평균 진도: 회당 8.4페이지
   
   **중간 섹션: 진도 그래프**
   - 날짜별 진도 누적 그래프 (꺾은선 그래프)
   - X축: 날짜, Y축: 페이지 수
   - 한눈에 진도 추세 파악 가능
   
   **하단 섹션: 수업별 진도 리스트**
   - 시간 역순으로 정렬
   - 각 항목:
     - 날짜
     - 진도 범위 (예: "53 ~ 67페이지")
     - 배운 내용 요약 (첫 줄만 미리보기)
     - 탭하면 해당 수업 기록 상세로 이동

4. 김선생님이 특정 수업 항목을 탭합니다.
   - 해당 수업의 전체 기록 확인 가능
   - "이 날 뭐 했지?" 궁금할 때 빠르게 찾기

5. 김선생님이 화면 우측 상단의 **"리포트 생성"** 버튼을 탭합니다.
   - 기간 선택 팝업: "최근 1개월", "최근 3개월", "전체", "직접 선택"
   - "최근 1개월"을 선택합니다

6. 진도 리포트가 생성됩니다:
   - 총 수업 횟수: 8회
   - 총 진도: 67페이지
   - 평균 진도: 회당 8.4페이지
   - 배운 주요 내용: 자동으로 수업 기록에서 키워드 추출하여 나열
   - **"학부모에게 공유"** 버튼

7. 김선생님이 **"학부모에게 공유"** 버튼을 누릅니다.
   - 확인 팝업: "이학생 학부모님께 한 달 진도 리포트를 공유할까요?"
   - **"확인"**을 탭합니다
   - 박학부모에게 알림 전송: "이학생님의 한 달 진도 리포트가 도착했습니다"

**결과**:
- 선생님은 한 달간의 수업 내용을 한눈에 파악
- 상담 시 구체적인 데이터 기반으로 대화 가능
- 학부모는 정기적인 진도 리포트를 받아 자녀의 학습 상황을 명확히 인지

### 시나리오 5: 선생님이 지난 수업 기록을 수정하는 경우

**배경**: 
김선생님이 어제 수업 기록을 작성했는데, "학생 상태" 부분에 오타가 있다는 것을 오늘 발견했습니다. 수정하려고 합니다.

**전제 조건**:
- 김선생님은 이미 로그인되어 있음
- 어제 작성한 수업 기록이 있음
- 이미 학부모에게 공유된 상태

**단계**:

1. 김선생님이 "이학생 수학 과외" 그룹 상세 화면에서 **"수업 기록" 탭**을 선택합니다.

2. 수업 기록 리스트에서 어제 날짜의 기록을 탭합니다.
   - 수업 기록 상세 화면으로 이동

3. 화면 우측 상단의 **"수정"** 버튼(연필 아이콘)을 탭합니다.
   - 수업 기록 편집 화면으로 전환
   - 기존 내용이 모두 입력 폼에 채워져 있음

4. 김선생님이 "학생 상태" 섹션의 오타를 수정합니다.
   - 기존: "오눌 집중력이 좋았어요"
   - 수정: "오늘 집중력이 좋았어요"

5. 화면 하단의 **"수정 완료"** 버튼을 탭합니다.

6. 확인 팝업이 나타납니다:
   - "수업 기록을 수정할까요?"
   - "학부모님께는 '수정됨' 알림이 전송됩니다."
   - **"확인"**, **"취소"** 버튼

7. 김선생님이 **"확인"**을 누릅니다.
   - 성공 메시지: "수업 기록이 수정되었습니다"
   - 수업 기록 상세 화면으로 돌아감
   - 화면 상단에 "수정됨" 뱃지가 표시됨

**결과**:
- 수업 기록이 수정되어 오타가 없어짐
- 학부모는 알림을 받아 "수정된 내용이 있구나" 인지 가능
- 기록 우측 하단에 "최종 수정: 2025.11.5 오전 10:30" 타임스탬프 표시
- 수정 이력이 자동으로 저장됨 (관리자 확인용)

- **숙제 (선택)**:
  - 최대 1000자
  - 이유: 구체적 지시 가능하되 과도하지 않게

- **교재명**:
  - 최소 1자, 최대 100자
  - 중복 허용
  - 이유: "수학의 정석 상권", "수학의 정석 하권" 등 유연하게 대응

- **진도 페이지**:
  - 1 이상의 정수
  - 시작 페이지 ≤ 끝 페이지
  - 이유: 논리적 유효성

### 규칙 그룹 2: 시간 제약사항

- **수업 기록 작성 가능 시간**:
  - 수업 시작 시간 이후부터 가능
  - 미래 수업에 대한 기록은 작성 불가
  - 이유: 실제 진행된 수업만 기록

- **수업 기록 수정 가능 기간**:
  - 작성 후 30일 이내만 수정 가능
  - 30일 초과 시 읽기 전용
  - 이유: 정산 완료 후 데이터 변경 방지

- **임시 저장 유효 기간**:
  - 7일
  - 7일 후 자동 삭제 및 알림
  - 이유: 오래된 임시 데이터 정리

### 규칙 그룹 3: 공유 및 알림 규칙

- **자동 알림 대상**:
  - 학생 본인: 항상 알림
  - 학부모: 항상 알림
  - 예외: 사용자가 푸시 알림 꺼둔 경우 앱 내 알림만

- **알림 전송 시점**:
  - 수업 기록 저장 버튼 누른 즉시
  - 최대 3회 재시도 (실패 시)
  - 재시도 간격: 1분, 5분, 30분

- **리포트 공유 권한**:
  - 선생님만 리포트 생성 및 공유 가능
  - 학부모/학생은 공유받은 리포트만 조회 가능
  - 이유: 선생님이 적절한 시점에 공유

- **수업 기록 수정 알림**:
  - 수정 시 학부모에게 "수정됨" 알림 전송
  - 수정 내역은 시스템 로그에 기록
  - 이유: 투명성 유지, 사후 분쟁 대비

### 규칙 그룹 4: 진도 계산 규칙

- **진도율 계산**:
  - (현재 페이지 / 전체 페이지) × 100
  - 소수점 첫째 자리까지 표시 (예: 27.9%)
  - 전체 페이지 미입력 시 "진도율 계산 불가" 표시

- **평균 진도 계산**:
  - 총 진행 페이지 / 수업 횟수
  - 소수점 첫째 자리까지 표시
  - 예: 67페이지 ÷ 8회 = 8.4페이지/회

- **진도 누적 기준**:
  - 교재별로 독립적으로 누적
  - 예: 교재 A는 30%, 교재 B는 50% 별도 관리
  - 이유: 여러 교재 병행 사용 가능

- **다중 교재 진도 입력**:
  - 한 수업에서 여러 교재의 진도를 기록할 수 있음
  - 예: 수학 문제집 50~55페이지 + 영문법 교재 20~25페이지
  - 각 교재별로 독립적으로 진도가 누적됨
  - 진도 추가는 "+ 다른 교재 진도 추가" 버튼으로 가능
  - 최대 5개 교재까지 한 수업에서 기록 가능
  - 이유: 실제 과외에서 여러 교재를 병행하는 경우 많음

### 규칙 그룹 5: 데이터 보존 규칙

- **수업 기록 삭제 제한**:
  - 작성 후 24시간 이내만 삭제 가능
  - 24시간 경과 후엔 삭제 불가, 수정만 가능
  - 이유: 학부모가 이미 본 기록이 갑자기 사라지면 신뢰 문제

- **교재 삭제 제한**:
  - 진도 기록이 있는 교재는 삭제 불가
  - "숨기기" 기능으로 목록에서만 제외
  - 이유: 과거 데이터 무결성 유지

- **그룹 삭제 시 처리**:
  - 그룹 삭제 시 수업 기록은 모두 보존 (읽기 전용)
  - 삭제된 그룹의 데이터는 90일간 복구 가능
  - 이유: 실수로 삭제했을 때 복구 가능성

## 6. UI 요구사항

### 화면 1: 수업 기록 작성 화면

**레이아웃**:
- 상단: 그룹명, 과목, 날짜, 시간 (읽기 전용, 자동 표시)
- 중앙: 입력 폼 (스크롤 가능)
- 하단: "임시 저장", "저장 및 공유" 버튼 고정

**주요 컴포넌트**:
- **오늘 배운 내용 입력창** (필수):
  - 큰 텍스트 에리어 (최소 5줄 높이)
  - 플레이스홀더 연한 회색
  - 글자 수 카운터 하단 우측 (예: "125 / 2000")
  - 10자 미만 시 하단에 경고 "조금 더 자세히 작성해주세요"
  
- **진도 입력 섹션** (선택):
  - 제목: "오늘 진행한 진도"
  - 교재 선택 드롭다운 (현재 사용 중인 교재 목록)
  - "+ 새 교재 등록" 버튼 (드롭다운 하단)
  - 시작 페이지, 끝 페이지 숫자 입력 (나란히 배치)
  - 자동 계산된 진도량 표시 (예: "14페이지 진행")
  - **"+ 다른 교재 진도 추가" 버튼**: 
    - 한 수업에서 여러 교재 진도 입력 가능
    - 클릭 시 추가 교재 진도 입력 필드 생성
    - 최대 5개 교재까지 추가 가능
    - 각 교재별로 삭제 버튼(X) 제공
  
- **학생 상태 입력창** (선택):
  - 중간 크기 텍스트 에리어 (3줄 높이)
  - 플레이스홀더: "예) 오늘 집중력이 좋았어요..."
  - 글자 수 카운터 "0 / 500"
  
- **숙제 입력창** (선택):
  - 중간 크기 텍스트 에리어 (3줄 높이)
  - 플레이스홀더: "예) 교과서 67~70페이지..."
  - 알림 아이콘 + "학생에게 알림이 전송됩니다"

**버튼**:
- **임시 저장**: 회색, 하단 좌측
- **저장 및 공유**: 파란색, 하단 우측 (더 크게)

**인터랙션**:
- 필수 항목 미입력 시 "저장 및 공유" 버튼 비활성화
- 입력 중 3분마다 자동 임시 저장 (상단에 "자동 저장됨" 작은 알림)
- 뒤로가기 버튼 누르면 "작성 중인 내용이 있어요. 임시 저장할까요?" 확인

### 화면 2: 수업 기록 상세 화면 (학부모/학생 뷰)

**레이아웃**:
- 상단: 날짜, 과목 뱃지, 출결 상태, 수업 시간
- 중앙: 수업 기록 내용 섹션 (스크롤)
- 하단: "공유하기" 버튼 (학부모만)

**주요 컴포넌트**:
- **섹션 1: 오늘 배운 내용**:
  - 제목 "오늘 배운 내용" (볼드)
  - 내용 텍스트 (여러 줄, 읽기 쉬운 간격)
  
- **섹션 2: 진도**:
  - 제목 "진도" (볼드)
  - 교재명 (배경색 있는 칩 형태)
  - 진도 범위 "53 ~ 67페이지"
  - 진행량 "14페이지 진행"
  - 전체 진도 "240페이지 중 67페이지 (27.9%)" (작은 글씨)
  
- **섹션 3: 학생 상태**:
  - 제목 "학생 상태" (볼드)
  - 내용 텍스트 (여러 줄)
  
- **섹션 4: 숙제**:
  - 제목 "숙제" (볼드)
  - 내용 텍스트
  - 완료 체크박스 (학생만 활성화)

**인터랙션**:
- 학부모가 처음 조회 시 "읽음" 상태 자동 기록
- 섹션 터치하면 확대 모드 (해당 섹션만 크게)
- 공유 버튼 누르면 옵션 시트: "카카오톡", "이미지 저장", "링크 복사"

### 화면 3: 진도 히스토리 화면

**레이아웃**:
- 상단: 교재 선택 탭 (수평 스크롤)
- 중간: 진도율 요약 카드 (큰 원형 진도 바)
- 중하단: 진도 그래프 (꺾은선)
- 하단: 수업별 진도 리스트 (스크롤)

**주요 컴포넌트**:
- **교재 탭**:
  - 각 교재명이 탭 (선택된 교재는 밑줄)
  - 좌우 스크롤 가능
  
- **진도율 요약 카드**:
  - 큰 원형 프로그레스 바 (가운데 퍼센트 표시)
  - 하단에 "67 / 240 페이지"
  - "평균 진도: 8.4페이지/회"
  
- **진도 그래프**:
  - X축: 날짜 (압축 표시, 예: "10/1", "10/15")
  - Y축: 페이지 수
  - 꺾은선으로 누적 진도 표시
  - 터치하면 해당 날짜 진도 툴팁
  
- **수업별 진도 리스트**:
  - 각 항목: 날짜, 진도 범위, 내용 미리보기 (첫 줄)
  - 탭하면 수업 기록 상세로 이동
  - 구분선으로 명확히 분리

**인터랙션**:
- 우측 상단 "리포트 생성" 버튼 (선생님만)
- 그래프 핀치 줌 (확대/축소)
- 리스트 새로고침 (당겨서 새로고침)

### 화면 4: 진도 리포트 화면

**레이아웃**:
- 상단: 기간 표시 (예: "2025.10.1 ~ 2025.11.4")
- 중앙: 요약 통계 카드들
- 하단: "학부모에게 공유" 버튼 (선생님만)

**주요 컴포넌트**:
- **통계 카드 1: 수업 횟수**:
  - 큰 숫자 "8회"
  - 작은 글씨 "총 수업 시간: 16시간"
  
- **통계 카드 2: 진도**:
  - 큰 숫자 "67페이지"
  - 작은 글씨 "평균: 8.4페이지/회"
  
- **통계 카드 3: 출석률**:
  - 큰 숫자 "100%"
  - 작은 글씨 "출석 8회, 결석 0회"
  
- **통계 카드 4: 주요 학습 내용**:
  - 키워드 추출 (예: #이차방정식, #판별식, #그래프)
  - 최대 10개 키워드

**인터랙션**:
- 각 카드 탭하면 상세 데이터로 이동
- "학부모에게 공유" 버튼 누르면 확인 팝업
- 공유 후엔 "이미 공유됨" 상태로 변경

## 7. 다른 기능과의 연결

### 의존하는 기능 (이것들이 먼저 있어야 함)

- **F-001: 회원가입 및 로그인** 
  - 이유: 사용자 인증 없이는 수업 기록 작성/조회 불가
  
- **F-002: 과외 그룹 생성 및 매칭** 
  - 이유: 그룹이 있어야 해당 그룹의 수업 기록 작성 가능. 학생/학부모 연결 필요
  
- **F-003: 수업 일정 관리** 
  - 이유: 일정이 있어야 "어느 수업에 대한 기록인지" 명확. 일정과 수업 기록 1:1 매핑
  
- **F-004: 출결 관리** 
  - 이유: 출결 체크된 일정에만 수업 기록 작성 가능. 출석한 수업만 기록하도록 제약

### 이 기능에 의존하는 기능 (이것 다음에 만들어질 것들)

- **F-006: 과외비 자동 정산** 
  - 이유: 수업 기록이 있어야 "실제 수업이 진행되었다"는 증거. 정산 분쟁 시 근거 자료
  
- **F-009: 숙제 관리** (2단계)
  - 이유: 수업 기록의 "숙제" 항목이 F-009의 숙제 목록으로 자동 연결. 이 기능이 기초 데이터 제공
  
- **F-010: 학습 대시보드** (2단계)
  - 이유: 수업 기록, 진도 데이터가 누적되어야 대시보드의 차트와 통계 생성 가능

### 함께 사용되는 기능 (밀접하게 연결됨)

- **F-008: 필수 알림 시스템** 
  - 관계: 수업 기록 작성 시 학생/학부모에게 자동 알림 전송. 리포트 공유 시에도 알림 발송
  
- **F-004: 출결 관리** 
  - 관계: 출결과 수업 기록이 같은 일정에 매핑됨. "오늘의 수업" 화면에서 출결 체크 → 수업 기록 작성 순서로 자연스럽게 연결

## 8. 향후 고려사항 (2단계 이후)

### 기능 확장

- **수업 기록 템플릿 저장**:
  - 선생님이 자주 쓰는 형식을 템플릿으로 저장 (예: "정규 수업 템플릿", "시험 대비 수업 템플릿")
  - 다음 수업 기록 작성 시 템플릿 불러와서 일부만 수정
  - 왜 2단계인가: MVP에서는 "지난 수업 복사" 기능만으로 충분. 템플릿은 고급 사용자용

- **사진/파일 첨부 기능**:
  - 수업 중 칠판 사진, 문제 풀이 사진, PDF 자료 첨부
  - 왜 2단계인가: 저장 용량 비용 발생. 초기엔 텍스트만으로 검증 후 확장

- **음성 입력 지원**:
  - 수업 직후 바쁜 선생님을 위해 음성으로 내용 녹음 → 자동 텍스트 변환
  - 왜 2단계인가: 음성 인식 정확도 문제, 추가 개발 시간 필요

- **학생 자가 평가 기능**:
  - 학생이 스스로 "오늘 이해도", "집중도" 등을 별점으로 평가
  - 선생님의 평가와 학생의 자가 평가를 비교하여 인사이트 제공
  - 왜 2단계인가: 학생의 자발적 참여가 필요. MVP에서 학생 사용 습관 형성 후 도입

### 개선 아이디어

- **AI 기반 수업 내용 요약**:
  - 선생님이 긴 내용을 입력하면 AI가 3줄 요약 생성 → 학부모에게 더 간결하게 전달
  - 왜 나중인가: AI API 비용, 정확도 검증 필요

- **진도 예측 기능**:
  - 현재 진도율과 남은 수업 횟수를 기반으로 "목표 완료일" 자동 계산
  - "현재 속도로 진행하면 3월 15일에 교재 완료 예상"
  - 왜 나중인가: 복잡한 계산 로직, 사용자 피드백 받은 후 개발

- **수업 기록 비교 기능**:
  - 여러 학생의 같은 기간 수업 기록을 나란히 비교 (선생님용)
  - "이 학생은 이차방정식 3회 만에 이해, 저 학생은 5회 소요" 등 인사이트
  - 왜 나중인가: UI 복잡도 증가, 개인정보 이슈 고려 필요

- **댓글/피드백 기능**:
  - 학부모나 학생이 수업 기록에 댓글로 질문이나 피드백 남기기
  - 선생님이 답변하여 양방향 소통
  - 왜 나중인가: 채팅 기능(F-014)과 중복 가능성, 우선순위 낮음

## 9. 성공 지표

### 정량적 지표

| 지표명 | 측정 방법 | 목표값 | 측정 시점 |
|--------|---------|--------|----------|
| 수업 기록 작성률 | (기록 작성된 수업 수 / 전체 완료된 수업 수) × 100 | 80% 이상 | 매주 |
| 평균 작성 소요 시간 | 수업 기록 작성 시작부터 저장까지 시간 | 3분 이내 | 매월 |
| 학부모 조회율 | (수업 기록 조회한 학부모 수 / 알림 받은 학부모 수) × 100 | 70% 이상 | 매주 |
| 평균 조회 시간 | 알림 받은 후 얼마나 빨리 조회하는지 | 6시간 이내 | 매월 |
| 진도 리포트 공유율 | 활성 선생님 중 월 1회 이상 리포트 공유 비율 | 50% 이상 | 매월 |
| 교재 진도율 완료 비율 | 교재 100% 완료한 케이스 비율 | 30% 이상 | 분기별 |

### 정성적 지표

- **사용자 피드백**: 
  - 기대: "수업 기록 작성이 간편해서 매번 쓰게 돼요", "학부모님께 자동으로 공유되니 따로 메시지 보낼 필요 없어서 좋아요"
  
- **문제 해결도**: 
  - 문제 3 (학부모와의 진도 공유 어려움) 80% 이상 해결
  - 측정: "진도를 학부모에게 공유하는 데 몇 분 걸리나요?" 설문 → "1분 이내" 응답 목표

- **신뢰도 향상**: 
  - "선생님이 우리 아이에게 관심이 있는 것 같아요" 학부모 응답 80% 이상 (5점 만점 중 4점 이상)

## 10. 알려진 제약사항 및 트레이드오프

### 기술적 제약

- **텍스트 기반 입력의 한계**: 
  - 초기 버전은 텍스트만 지원. 이미지나 파일 첨부 불가
  - 이유: 저장 용량 비용 절감, 빠른 MVP 출시 우선
  - 트레이드오프: 일부 선생님은 칠판 사진 등을 첨부하고 싶어할 수 있음. 2단계에서 추가 예정

- **오프라인 작성 제한**:
  - 네트워크 없으면 임시 저장만 가능, 학부모 공유는 온라인 복구 후
  - 이유: 실시간 알림 전송을 위해 서버 통신 필수
  - 트레이드오프: 지하 강의실 등에서 바로 작성 어려움. 로컬 저장 후 자동 동기화로 보완

### 의도적 선택

- **출결 체크 없이 수업 기록 작성 불가**:
  - 포기한 유연성: 출결은 안 했지만 수업 내용만 기록하고 싶은 경우
  - 얻은 것: 데이터 정합성. 출결과 수업 기록이 항상 짝을 이루어 정산 오류 방지
  - 이유: "결석인데 수업 기록이 있다" 같은 논리적 모순 차단

- **30일 이후 수업 기록 수정 불가**:
  - 포기한 유연성: 한참 후에 오타 발견해도 수정 못함
  - 얻은 것: 정산 완료 후 데이터 변조 방지. 학부모-선생님 간 분쟁 시 신뢰할 수 있는 기록
  - 이유: 금전 거래와 연결된 기록의 무결성이 더 중요

- **진도는 교재 단위로만 관리**:
  - 포기한 기능: "단원별 진도", "챕터별 진도" 등 세밀한 분류
  - 얻은 것: 단순한 UX. 선생님이 복잡한 설정 없이 바로 사용 가능
  - 이유: 과외는 학원과 달리 교재가 명확. 페이지 번호만으로 충분

- **숙제는 텍스트만, 제출 시스템 없음**:
  - 포기한 기능: 숙제 파일 업로드, 제출 확인 시스템
  - 얻은 것: 개발 리소스 절약, 빠른 출시
  - 이유: MVP에서는 "숙제가 뭔지 알려주는 것"만으로 충분. 제출 관리는 F-009(2단계)에서
